# HestiaCP & Debian 12 System Paths Map

This reference document maps the verified system paths for HestiaCP running on Debian 12. It serves as the primary source of truth for all diagnostic and configuration tasks.

## 1. HestiaCP & Debian Core
| Component | Path | Description |
| :--- | :--- | :--- |
| **Hestia Base** | `/usr/local/hestia` | Installation root |
| **Hestia Binaries** | `/usr/local/hestia/bin` | CLI commands (`v-*`) |
| **Hestia Config** | `/usr/local/hestia/conf/hestia.conf` | Global configuration |
| **Debian Release** | `/etc/debian_version` | OS Version (Bookworm 12.x) |
| **OS Release** | `/etc/os-release` | Distribution details |
| **User Data** | `/usr/local/hestia/data/users` | Flat-file user database |
| **Packages** | `/usr/local/hestia/data/packages` | User packages (plans) |
| **Queue** | `/usr/local/hestia/data/queue` | Task queue (backup, restore, web) |
| **IPs** | `/usr/local/hestia/data/ips` | IP address configuration |
| **Sessions** | `/usr/local/hestia/data/sessions` | HestiaCP Panel sessions (Login debug) |
| **Install Scripts** | `/usr/local/hestia/install/` | Installation and upgrade scripts |
| **Panel Logs** | `/var/log/hestia/system.log` | Panel action audit log |
| **Panel Nginx Logs** | `/var/log/hestia/nginx-error.log` | Panel web server errors (Port 8083) |
| **Backup Logs** | `/var/log/hestia/backup.log` | Backup history log |

## 13. Virtual Paths & Logic (Meta-Info)
Logic to retrieve dynamic information not found in static files.

| Info | Retrieval Method | Example Command |
| :--- | :--- | :--- |
| **Admin Email** | Read `CONTACT` in admin's `user.conf` | `grep "^CONTACT=" /usr/local/hestia/data/users/admin/user.conf \| cut -d"'" -f2` |
| **User Home** | Read `/etc/passwd` or `hestia.conf` | `grep "^HOME=" /usr/local/hestia/conf/hestia.conf \| cut -f 2 -d \'` |
| **Git Dest** | Read `github-mirror.conf` | `grep "^$USER|" /etc/hestiacp-github-mirror.conf \| cut -d"|" -f4` |
| **PHP Processes** | Count FPM processes per pool | `pgrep -c 'php-fpm: pool www'` |
| **PHP Crashes** | Count zombie processes | `ps aux \| grep '[p]hp-fpm' \| awk '$8 ~ /Z/ {count++} END {print count+0}'` |

## 14. Known False Paths (Do Not Use)
These are common pitfalls for generic Linux but invalid for HestiaCP.

| Generic Path | Hestia Reality | Why? |
| :--- | :--- | :--- |
| `/etc/letsencrypt/live` | `/home/[USER]/conf/web/[DOMAIN]/ssl/` | Hestia manages SSL per-user, not globally. |
| `/etc/apache2/sites-enabled` | `/etc/apache2/conf.d/domains/` | Hestia includes user configs dynamically. |
| `/etc/nginx/sites-enabled` | `/etc/nginx/conf.d/domains/` | Same as Apache. |
| `/var/www/html` | `/home/[USER]/web/[DOMAIN]/public_html` | Hestia is multi-user by design. |
